Quick Menu Setup

To start using this menu, there are 4 necessary files :

  • css/mgmenu.css
  • js/jquery.js
  • js/mgmenu_plugins.js
  • js/mgmenu.js

In your HTML files, you have to make sure that you link properly to each of them.

You'll also find at the top of each HTML file a small script, it is required and it's the place where you can set up a few options that we'll cover later. Just keep in mind that this small script must be placed somewhere after the megamenu script file (or if you want you can move it at the end of mgmenu.js).

This script must be repeated for each menu that appears on a same page. For example if you include 2 menus on a same page, this script should be there twice (3 times if you have 3 menus, etc.).

The folder img contains a few graphics that can be used in the menu, essentially icons but they're totally optionnal.

Once you have the javascript, CSS and image files in place, you copy and paste a menu example. In all HTML files provided in this package, the menu is clearly delimited so you just have to take all the code between :

<-- THE MENU STARTS HERE -->

And

<-- THE MENU ENDS HERE -->

And that's it, if everything went fine, the menu should be in place. Let's finish with a couple of options : in all HTML files provided, you can see that the menu starts with a code like this :

<div id="mgmenu1" class="mgmenu_fixed">

First you can see that the menu has a unique ID and it's referenced in the top script. By providing a unique ID, you can place several menus on a same page and they're handeled separately by the jquery script.

Then you have a class and here you have a choice between 3 options :

  • mgmenu_fixed which sticks the menu at the top of the browser
  • mgmenu_footer which turns the menu into a sticky footer
  • mgmenu_container which makes a regular mega menu not fixed anywhere on the page

And you can switch between any of these 3 variants just by modifying a single class !

Menu Effects, Javascript Files

If you read the previous section, you should know how to basically integrate a mega menu into your HTML pages. It's now necessary to have a look to the scripts, it's not required to have some coding knowledge but you can learn here how to safely change a few options.

First of all, if you add this menu in an existing website, there are chances that jQuery is already there. If it's the case, you don't need to add it again for this menu, but you may want to check the version already in place and possibly upgrade to a newer version. In all cases, it's recommended to have a single instance of jQuery otherwise you may encounter issues.

This mega menu as I said earlier requires 3 javascript files :

  • jquery.js
  • mgmenu_plugins.js
  • mgmenu.js

So in your header you should have something like this :

<script src="js/jquery.js"></script>
<script src="js/mgmenu_plugins.js"></script>
<script src="js/mgmenu.js"></script>

Right after, it's the place where you should have the small script used to set up the menu options and it looks like this :

<script>
$(document).ready(function($){
    $('#mgmenu1').universalMegaMenu({
        menu_effect: 'hover_fade',
        menu_speed_show: 300,
        menu_speed_hide: 200,
        menu_speed_delay: 200,
        menu_click_outside: true,
        menubar_trigger : true,
        menubar_hide : false,
        menu_responsive: true
    });
    megaMenuContactForm(); 
});
</script>

In this script there's an easy change you can try : it's the option menu_effect. You can try the following values :

  • hover_fade : fade effect on mouse hover
  • hover_slide : slide effect on mouse hover
  • click_fade : the drop down fades in on mouse click (and remains visible as long as the mouse hovers on it)
  • click_slide : the drop down slides on mouse click (and remains visible as long as the mouse hovers on it)
  • open_close_fade : 1 click to open, 1 click to close (fade effect)
  • open_close_slide : 1 click to open, 1 click to close (slide effect)

Besides the menu effect, you have a few more options that can be modified :

  • menu_speed_show : time in milliseconds to show the drop downs (the best is any number between 100 and 200)
  • menu_speed_hide : time in milliseconds to hide the drop downs
  • menu_speed_delay : time in milliseconds to wait before showing a drop down
  • menu_click_outside : closes any visible drop down if set to true (only when you use the menu effects open_close_fade or open_close_slide)
  • menubar_trigger : enables a button to show / hide the menu bar if set to true
  • menubar_hide : hides the menu bar when the page loads if set to true (and should be used along with the trigger button above)
  • menu_responsive : enables an optimized script to handle touch screen devices

There is an additional line of code that you can find in some HTML examples :

megaMenuContactForm();

This is required only if you use the contact form provided with the menu. If you use another contact form or if you don't want to use this one, then you just have to remove this above line of code.

Lastly, if you put for example 2 distinct menus on a same page, you'll need to reference to each of them with a unique ID, and each of them will have a separate setup. It means that you can choose different options like this :

<script>
$(document).ready(function($){
    $('#mgmenu1').universalMegaMenu({
        menu_effect: 'click_slide',
        menu_speed_show: 300,
        menu_speed_hide: 200,
        menu_speed_delay: 200,
        menu_click_outside: true,
        menubar_trigger : false,
        menubar_hide : false,
        menu_responsive: true
    });
    $('#mgmenu2').universalMegaMenu({
        menu_effect: 'open_close_fade',
        menu_speed_show: 300,
        menu_speed_hide: 200,
        menu_speed_delay: 200,
        menu_click_outside: true,
        menubar_trigger : true,
        menubar_hide : true,
        menu_responsive: true
    });
});
</script>
It's just an example and you can try different options as long as each menu has its own unique ID, this is really important.

Menu Bar

This menu will automatically expand to take all the available horizontal space. So if you place the menu into a 960px container, it will inherit those 960px. It works in the same way for percentages. In the example pages, you can see that I've put the menu into this container :

<div class="page_wrapper">

</div>

But this is a container that you don't need to use at all, it's not part of the menu and it's just here for showing purposes.

Now let's see a basic menu setup and start with the first wrapper :

<div id="mgmenu1" class="mgmenu_fixed">

</div>

As we have seen earlier, each separate menu (on a same page) should have a unique ID so let's say for example that you want a fixed menu at the top and a fixed footer, it'll look like this :

<div id="mgmenu1" class="mgmenu_fixed">

</div>

<div id="mgmenu2" class="mgmenu_footer">

</div>

For your menu you have a choice between 3 options :

  • mgmenu_fixed which sticks the menu at the top of the browser
  • mgmenu_footer which turns the menu into a sticky footer
  • mgmenu_container which makes a regular mega menu not fixed anywhere on the page

And you can easily switch between any of these 3 variants just by modifying a single class.

Now let's get back to our first example, we started the menu with this code :

<div id="mgmenu1" class="mgmenu_fixed">

</div>

From there we'll add the second container :

<div id="mgmenu1" class="mgmenu_fixed">

    <ul class="mgmenu">

    </ul>

<div id="mgmenu1" class="mgmenu_fixed">

    <ul class="mgmenu">

        <li><a href="#"><i class="mini_icon ic_tag"></i>Item 2</a></li>

Additionnally, if you use the responsive version of the menu, the whole menu bar is switched to a button on mobile devices. This functionnality requires this element :

<li class="mgmenu_button">Mega Menu</li>

<div id="mgmenu1" class="mgmenu_fixed">

I've put here 2 different examples to show you that these menu items can be links or span elements. The <i> element is also optionnal, just use it if you want an icon to appear on the left of the menu element (see the next section for more explanations).

Drop Down Containers

For more simplicity, we'll work with a simplified markup so let's take back our example to add a drop down :

<div id="mgmenu1" class="mgmenu_fixed">

    <ul class="mgmenu">

        <li><span><i class="mini_icon ic_bookmark"></i>Item 1</span></li>

    </ul>

</div>

The drop down will be placed within the LI element as following :

<div id="mgmenu1" class="mgmenu_fixed">

    <ul class="mgmenu">

        <li><span><i class="mini_icon ic_bookmark"></i>Item 1</span>

            <div class="dropdown_fullwidth">

            </div>

        </li>

    </ul>

</div>

Here we've added a DIV element that has one class : dropdown_fullwidth. It means that this drop down will have a width equal to the width of the menu bar.

As alternatives, you have 11 other sizes for the drop downs, and all of them are coded in this way :

<div id="mgmenu1" class="mgmenu_fixed">

    <ul class="mgmenu">

        <li><span><i class="mini_icon ic_bookmark"></i>Item 1</span>

            <div class="dropdown_container dropdown_4columns">

            </div>

        </li>

    </ul>

</div>

You first need the dropdown_container class which is used to reference any drop down other than a full width drop down. This class is followed by dropdown_4columns in this example but you have the choice between 11 sizes (ordered from the smallest to the largest) :

  • dropdown_1column
  • dropdown_2columns
  • dropdown_3columns
  • dropdown_4columns
  • dropdown_5columns
  • dropdown_6columns
  • dropdown_7columns
  • dropdown_8columns
  • dropdown_9columns
  • dropdown_10columns
  • dropdown_11columns

If you use dropdown_1column, you'll have the smallest size for a drop down, and on the opposite, dropdown_11columns will be the largest (without considering dropdown_fullwidth).

To sum this up, you either use a full width drop down with dropdown_fullwidth or a sized drop down by using dropdown_container followed by dropdown_Xcolumns where X is a number between 1 and 11.

Drop Down Columns

We have seen earlier how to set up containers for the drop downs. Now within any of these drop downs, you'll need to put your content organized in columns. It means that for any drop down, you can have between 1 and 12 columns of content that will scale automatically to fit. Let's use our previous markup and add a few columns :

<div id="mgmenu1" class="mgmenu_fixed">

    <ul class="mgmenu">

        <li><span><i class="mini_icon ic_bookmark"></i>Item 1</span>

            <div class="dropdown_container dropdown_4columns">

                <div class="col_12">
                Title
                </div>

                <div class="col_8">
                Left column
                </div>

                <div class="col_4">
                Right column
                </div>

            </div>

        </li>

    </ul>

</div>

You have the choice between 12 column sizes, from the largest to the smallest :

col_12 is the largest and col_1 the smallest. Now you have to keep in mind that to create a row of content, you need to reach a value of 12 by adding the numbers used in these class names. For example, col_12 will expand to the whole drop down space available. If you add other columns, they'll fall under this one.

In the sample above, I've used col_12 and the second row of content has col_8 combined with col_4 : 8 + 4 = 12, together they exand to the whole space available. These column containers are using percentages so you can think of it with this system :

  • col_12 is 100%
  • col_11 is 11/12 of 100%
  • col_10 is 10/12 of 100%
  • col_9 is 9/12 of 100%
  • col_8 is 8/12 of 100%
  • col_7 is 7/12 of 100%
  • col_6 is 6/12 of 100%
  • col_5 is 5/12 of 100%
  • col_4 is 4/12 of 100%
  • col_3 is 3/12 of 100%
  • col_2 is 2/12 of 100%
  • col_1 is 1/12 of 100%

Each time you build your columns, you need to reach 100% of the width by combining these columns (and adding the number representing them, like col_6 is half of the width, put it twice and you get 2 equal columns). Here are other examples :

This example above is more complex but should show you exactly how it works : we have seen that columns are proportionnal and col_12 represents 100% of the width. Each time you reach a value of 12 by adding the column number values, it makes a row of content. Here's another example :

  • col_5 + col_7 is the first row (5 + 7 = 12)
  • col_6 + col_3 + col_3 is the second row (6 + 3 + 3 = 12)
  • col_3 + col_2 + col_2 + col_1 + col_4 is the third row (3 + 2 + 2 + 1 + 4 = 12)

You'll maybe need to time to understand how this system works, it's a little bit complicated but once you get the logic, things are much easier and incredibly flexible. Now you don't have to think about pixel values, all the elements are resized to fit into any context.

Fly-Out Menu

This menu includes a multi-level fly-out menu and you can add as many levels as you want but the script won't keep them inside the viewport : if you don't want some levels to fall outside the visible area, you have to manually set their alignment, they can be displayed towards the left or the right.

This fly-out menu is made of nested unordered lists and use the same jQuery effect as the mega drop downs. It is not possible to set a different effect for each type of drop down.

Here's the basic structure of a fly-out menu :

<div id="mgmenu1" class="mgmenu_fixed">

    <ul class="mgmenu">

        <li><span><i class="mini_icon ic_bookmark"></i>Fly-Out</span>

            <ul class="dropdown_flyout">

            </ul>

        </li>

    </ul>

</div>

For more simplicity, we'll keep only the code that starts with the UL element :

<ul class="dropdown_flyout">

</ul>
 
Here you can add simple LI element that can be links :
 
<ul class="dropdown_flyout">

    <li><a href="#">Link 1</a></li>
    <li><a href="#">Link 2</a></li>
    <li><a href="#">Link 3</a></li>

</ul>
Any of these elements can hold a sublevel like so (we'll add a sublevel to link 1) :
 
<ul class="dropdown_flyout">

    <li class="dropdown_parent"><a href="#">Link 1</a>

        <ul class="dropdown_flyout_level">
            <li><a href="#">Link 1-1</a></li>
            <li><a href="#">Link 1-2</a></li>
        </ul>

    </li>

    <li><a href="#">Link 2</a></li>
    <li><a href="#">Link 3</a></li>

</ul>
And we can go deeper by adding a sublevel to the link 1-1 as following :
 
<ul class="dropdown_flyout">

    <li class="dropdown_parent"><a href="#">Link 1</a>

        <ul class="dropdown_flyout_level">

            <li class="dropdown_parent"><a href="#">Link 1-1</a>

                <ul class="dropdown_flyout_level">

                    <li><a href="#">Link 1-1-1</a></li>
                    <li><a href="#">Link 1-1-2</a></li>
                    <li><a href="#">Link 1-1-3</a></li>

                </ul>

            </li>

            <li><a href="#">Link 1-2</a></li>

        </ul>

    </li>

    <li><a href="#">Link 2</a></li>
    <li><a href="#">Link 3</a></li>

</ul>
And this can continue as much as you want. There's an additional option allowing you to make any sublevel appear on the left side (instead on the right). From our previous example, let's make the last level of elements appear on the right :
 
<ul class="dropdown_flyout">

    <li class="dropdown_parent"><a href="#">Link 1</a>

        <ul class="dropdown_flyout_level">

            <li class="dropdown_parent dropdown_parent_left"><a href="#">Link 1-1</a>

                <ul class="dropdown_flyout_level dropdown_flyout_level_left">

                    <li><a href="#">Link 1-1-1</a></li>
                    <li><a href="#">Link 1-1-2</a></li>
                    <li><a href="#">Link 1-1-3</a></li>

                </ul>

            </li>

            <li><a href="#">Link 1-2</a></li>

        </ul>

    </li>

    <li><a href="#">Link 2</a></li>
    <li><a href="#">Link 3</a></li>

</ul>

For that, we've simply added dropdown_parent_left to the parent element and dropdown_flyout_level_left to the sublevel that should appear on the right.

This is a very basic example but it's just to understand how it works. The HTML pages provide a good example with a 3-levels fly-out menu.

Tabs

In the HTML pages provided in the package, you'll find that some drop downs use a system of tabs or panels. You can have them on several drop downs and they all follow the same structure : you have first the links (that load the corresponding part once clicked), and the panels.

These tabs can be used in any type of drop down, full width or not. As an example, we'll take back the code used to explain how to set up a basic drop down :

<div id="mgmenu1" class="mgmenu_fixed">

    <ul class="mgmenu">

        <li><span><i class="mini_icon ic_bookmark"></i>Item 1</span>

            <div class="dropdown_fullwidth">

            </div>

        </li>

    </ul>

</div>

For more simplicity, we'll keep only the drop down code :

<div class="dropdown_fullwidth">

</div>

So here we need to add the mgmenu_tabs as following :

<div class="dropdown_fullwidth mgmenu_tabs">

</div>

From there we can add 2 links that will be the navigation system :

<div class="dropdown_fullwidth mgmenu_tabs">

    <ul class="mgmenu_tabs_nav">

	    <li><a href="#section1" class="current">Tab 1</a></li>
	    <li><a href="#section2">Tab 2</a></li>

    </ul>

</div>

Each link must point to a unique anchor, for example if you use several tab systems in different drop downs, you should make sure that each one has its own anchor / ID. Also each section, except the first one, must have the class mgmenu_tabs_hide in order to hide them - so the first one remains visible.

After the links, you can put the containers that will hold the tabs :

<ul class="mgmenu_tabs_nav">

	<li><a href="#section1" class="current">Tab 1</a></li>
	<li><a href="#section2">Tab 2</a></li>
	<li><a href="#section3">Tab 3</a></li>

</ul>

<div class="mgmenu_tabs_panels">

	<div id="section1">

	</div>

	<div id="section2" class="mgmenu_tabs_hide">

	</div>

	<div id="section3" class="mgmenu_tabs_hide">

	</div>

</div>

And that's it, you have all the necessary markup to hold a system of 2 tabs. You can use the column system described above within those tabs.

Additionnally, the tabs links can be place on the left side (by default) or on any other side by using the following classes :

  • mgmenu_tabs_top
  • mgmenu_tabs_bottom
  • mgmenu_tabs_right

If we take our previous example and want to place the link to the top, the code will be :

<div class="dropdown_fullwidth mgmenu_tabs mgmenu_tabs_top">

...

</div>

CSS Files and Structure

In the css folder, you'll find the only file required : mgmenu.css. It contains all the required (and optional) styling for the mega menu.

If you want to modify some properties in the CSS, make sure that you keep a safe copy of the original file before starting. This CSS is quite complex and requires a good knowledge of CSS markup to be safely modified.

This file is divided into 8 main parts and modifying this file may affect the functionnality so be very careful when you start your customizations :

00 PAGE STYLING (REMOVABLE)
01 MENU BAR
   1. Mega Menu Bar
   2. Mega Menu Trigger Button
02 DROP DOWNS
	1. Common Styling
	2. Footer
	3. Drop Down Containers
03 COLUMNS GRID
04 MULTILEVEL DROP DOWN
05 CONTENT & TYPOGRAPHY
   1. Basic Typography 
   2. Lists
   3. Other Styling
   4. Inline Images
   5. Icons
   6. Form Elements
06 DROP DOWN TABS
07 MOBILE DEVICES
08 RETINA DISPLAYS

Here are additional explanations of the content of each part of the CSS :

  • 01 MENU BAR : It's the code for the menu container and the menu items (the entire menu bar without drop downs).
  • 02 DROP DOWNS : This part defines all the drop down containers that will hold the columns.
  • 03 COLUMNS GRID : Here you'll find the column grid system, everything is in percentages.
  • 04 MULTILEVEL DROP DOWN : Instead of mega drop downs, you can use regular fly-out menus with infinite sub-levels.
  • 05 CONTENT & TYPOGRAPHY : This part contains styling for elements like headers, lists, forms, images, social icons, etc.
  • 06 DROP DOWN TABS : This is the styling of the tabs used within the drop downs.
  • 07 MOBILE DEVICES : Here are all the necessary media queries making this menu completely responsive.
  • 08 RETINA DISPLAYS : This is the code that loads optimized graphics for retina displays.

The menu appears behind another element (a slider for example), how to fix that ?

Most of the time it's a simple z-index issue which you can think of the way to stack element on top of each other. By default, the menu has a z-index of 12 so if you have a slider with a z-index of 13 (or any value more than that), the menu will fall behind this slider.

And the opposite is true, for example if you have a lightbox with a z-index of 10, it'll appear behind the menu. Here's an illustrative case : on your page, you have this menu, a slider and a lightbox. The lightbox should always be on top of everything else so it should have the highest z-index. The menu should be above the slider but under the lightbox and finally the slider should appear behind both the menu drop downs and the lightbox. The 3 z-index values should follow this order : lightbox > menu > slider

That's it, so if you have this kind of issue, there are chances that your other elements have a z-index aswell and you should check the existing values.

Lastly, if you manually add z-index values, keep in mind that they are applied to an element only if it's a positionned element. It can be relative, absolute or fixed, the z-index works along with a defined position.
